# Week 6: Neural Network American Put Pricer

A PyTorch MLP trained to imitate the Week 4 CRR binomial American put pricer
(`../binomial-tree/american_put.py`). Generates synthetic contracts, labels them with the
binomial baseline, trains a small neural network, and checks whether the network is
financially sensible — not just low average error.

## Quick Start

### Setup
```bash
pip install numpy torch matplotlib pytest
```

### Run the full pipeline
```bash
python pipeline.py
```
This regenerates the dataset, trains the model, and writes:
- `artifacts/week6_option_data.npz` — sampled contracts + binomial labels
- `artifacts/week6_neural_pricer.pt` — model weights + scaling stats + metadata
- `figures/learning_curve.png`, `figures/pred_vs_binomial.png`, `figures/surface_comparison.png`

### Run the report notebook
```bash
jupyter notebook week6_report.ipynb
```
Same pipeline, cell by cell, with narrative and inline plots.

### Run the tests
```bash
pytest test_neural_pricer.py -v
```

## Clean-session test

Restart Python, then:
```python
import torch
from model import PutPricerNet, load_artifact
import numpy as np

loaded = load_artifact("artifacts/week6_neural_pricer.pt")
model = PutPricerNet(input_dim=5, hidden=128)
model.load_state_dict(loaded["model_state"])
model.eval()

x_mean, x_std = loaded["x_mean"], loaded["x_std"]
x = np.array([[100.0, 100.0, 1.0, 0.05, 0.25]])
x_scaled = (x - x_mean) / x_std
price = model(torch.tensor(x_scaled, dtype=torch.float32)).item()
print(price)  # should be a sane put price, in the same ballpark as crr_put_price(100,100,1,0.05,0.25,500)
```

## Sampled ranges (fixed, documented)

| Feature | Range |
|---|---|
| S0 | 60 to 140 |
| K | 80 to 120 |
| T (years) | 0.05 to 2.0 |
| r | 0.00 to 0.10 |
| sigma | 0.10 to 0.50 |

Labels use a fixed binomial step count of **500** everywhere (dataset generation and the
surface comparison), so the network never has to absorb discretization noise mixed in with
the true pricing shape.

## Architecture

`PutPricerNet`: 5 raw inputs (`S0, K, T, r, sigma`) → `Linear(5, 128) + ReLU` →
`Linear(128, 128) + ReLU` → `Linear(128, 1)`. No sigmoid on the output. Trained with Adam
(`lr=1e-3`), MSE loss, up to 300 epochs, keeping the best-validation-loss checkpoint (not the
final epoch).

## Files

- `data.py` — contract sampling, binomial labeling (via Week 4's `crr_put_price`), label sanity
  checks, dataset save/load.
- `split.py` — 80/10/10 train/val/test split; standardization fit on the training set only.
- `model.py` — `PutPricerNet` MLP.
- `train.py` — training loop with best-checkpoint selection; learning-curve plot.
- `evaluate.py` — MAE/RMSE/max-abs-error, moneyness-bucketed MAE, predicted-vs-binomial scatter.
- `surface.py` — NN-vs-binomial surface grid, finance sanity checks (monotonicity in S0,
  non-negativity, intrinsic-value bound), surface/error plots.
- `pipeline.py` — end-to-end CLI entry point tying everything together and saving the artifact.
- `test_neural_pricer.py` — pytest suite covering sampling ranges, label sanity, split/scaling
  leakage, model output shape, and the sanity-check logic itself.
- `week6_report.ipynb` — narrative notebook that imports and runs the modules above.
- `figures/` — saved plots.
- `artifacts/` — saved dataset and model artifact (regenerable via `pipeline.py`).

## Key Results

From the run currently reflected in `figures/` and `artifacts/` (generated by executing
`week6_report.ipynb` end to end): 12,000 contracts, 500 binomial steps, 80/10/10 split
(9600/1200/1200), 300 training epochs, val MSE ≈ 0.0094 by epoch 275.

| Metric | Value |
|---|---|
| Test MAE | 0.0634 |
| Test RMSE | 0.0853 |
| Test max abs error | 0.5608 |
| MAE — deep ITM put (S0/K < 0.85, n=364) | 0.0611 |
| MAE — near ATM (0.85–1.15, n=443) | 0.0769 |
| MAE — deep OTM put (S0/K > 1.15, n=393) | 0.0504 |
| Monotonicity violations (fixed S0 slice, K=100,T=1,r=5%,σ=25%) | 0 |
| Negative predictions (same slice) | 0 |
| Below-intrinsic violations (same slice, 81 points) | 10, max magnitude 0.061 |

Label range across the dataset: [0.00, 59.80], mean 14.81 — so a test MAE of ~0.063 is roughly
0.4% of the mean label, and the predicted-vs-binomial scatter (`figures/pred_vs_binomial.png`)
hugs the 45° line with no visible bias.

⚠️ **Note on reproducibility:** re-running `pipeline.py` or the notebook with the same seed
(42) gives numbers in this same ballpark but not bit-identical — PyTorch's CPU matmul
reduction order isn't deterministic across runs even with a fixed seed unless
`torch.use_deterministic_algorithms` and single-threaded execution are forced. The dataset
sampling and split (pure NumPy) are exactly reproducible; only the trained weights drift
slightly run to run. An earlier run of `pipeline.py` alone produced Test MAE 0.0647 and only 3
below-intrinsic violations — same conclusions, slightly different numbers.

**Where the model struggles most:** the near-ATM bucket has the highest MAE of the three
moneyness buckets, consistent with that region having the most curvature in the price surface
(closest to the early-exercise boundary). The surface error heatmap
(`figures/surface_comparison.png`) shows errors scattered mostly at short maturities
(T < 0.5) rather than concentrated in one contiguous region — the model doesn't have a single
systematic blind spot, but it is least precise where time value changes fastest. The
below-intrinsic violations on the sanity-check slice cluster at S0 = 68–77 (deep-ish ITM, short
maturity), with magnitudes up to 0.06 — small relative to prices in the tens, but a reminder
that the network is a close, not exact, approximation and is not guaranteed arbitrage-free.
